配置文件详解

Bilibili 直播弹幕助手 的配置文件是方便直播启动的配置文件，是一个JSON的配置文件。

如果您是新手，建议您使用 快速配置工具 生成配置文件。

结构

  • 一个基础的 config.json 是类似的 : 
    {
  "room": {
    "live_id": 19386,
    "live_url": "http://live.bilibili.com/19386"
  },
  "settings": {
    "showTime": false,
    "showUserName": true,
    "showWatcherNum": false,
    "show_gift": true,
    "output": true,
    "output_path": null,
    "output_filename": "%t.comments"
  },
  "server": {
    "version": 2
}
    由三部分组成，第一部分是直播间的配置（即room），第二部分是弹幕服务器客户端的配置（即settings），第三部分是弹幕服务器协议的配置（即server）。
    下面将会分开讲解各配置的意义。

弹幕服务器客户端(settings)

showTime 弹幕是否显示时间

  • [弹幕]是否显示时间? true 是 / false 否
    建议为 false
  • 如果设置为 true 那么控制台会显示成这样: 
      [2014-01-01 00:00:00] ...

showWatcherNum 是否显示在线人数

  • [服务器]是否显示在线人数? true 是 / false 否
    心跳包返回数据后显示。心跳包频率为 20s/次。
    建议为 false (除非需要实时观察人数)
  • 如果设置为 true 那么控制台会显示成这样: 
    [系统] 在线人数 1024

showUserName 弹幕是否显示发布者

  • [弹幕]是否显示发布者? true 是 / false 否
    建议为 true
  • 如果设置为 true 那么控制台会显示成这样: 
     ... ⑨bishi : ...

show_gift 是否显示礼物

  • [弹幕]是否显示礼物? true 是 / false 否
    建议为 true
  • 如果设置为 true 那么控制台会显示成这样: 
    [礼物] ⑨bishi赠送我宣你*999

output 是否保存弹幕源数据

  • [弹幕]是否保存弹幕源数据? true 是 / false 否
    建议为 false
  • 弹幕源数据默认保存至 程序根目录下的 comment 文件夹,命名格式 <comment>_<%t>.log
    具体弹幕源数据的格式请看弹幕源数据开发文档。

output_path 弹幕源数据的保存路径

  • [弹幕]弹幕源数据的保存路径
  • 弹幕源数据默认保存至 程序根目录下的 comment 文件夹

output_filename 弹幕源数据保存文件的格式

  • [弹幕]弹幕源数据保存文件的格式
  • 弹幕源数据默认的命名格式 <comment>_<%t>.log
    其中%t == unix timestamp

直播间(room)

概述

  • 本设置为必填项目，用于访问弹幕服务器。有live_id / live_url两个参数。
  • 只需要选择一个参数设置就可以访问弹幕服务器了。
  • 下面会讲到二者差别。

live_url 直播间网址

  • 直播间网址
    例如: http://live.bilibili.com/19386 (不支持ID小于1001的短号)

live_id 直播间ID

  • 直播间ID
    例如 19386

弹幕服务器协议(server)

概述

  • 本设置为选填项目，用于访问指定的弹幕服务器。有host / port / version三个参数。

host 弹幕服务器主机名

  • 弹幕服务器主机名
    例如: livecmt-1.bilibili.com

port 弹幕服务器端口

  • 弹幕服务器端口
    例如 788

version 弹幕服务器版本号

  • 弹幕服务器版本号
    例如 2
    暂时支持 1 和 2, 请勿随意设置!
