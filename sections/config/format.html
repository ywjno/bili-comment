配置文件详解

Bilibili 直播弹幕助手 的配置文件是根目录下的 config.js。

如果您是新手，建议您使用 快速配置工具 生成配置文件。

结构

  • 一个基础的 config.js 是长这样的 : 
    exports.config={
    "showTime":false,
    "showWatcherNum":true,
    "showUserName":true,
    "reconnect":true,
    "save":true
};
exports.liveroom={
    url:'',
    roomid:null
};
    由两部分组成，第一部分是弹幕服务器客户端的配置（即exports.config），第二部分是直播间的配置（即exports.liveroom）。
    下面将会分开讲解各配置的意义。

弹幕服务器客户端(exports.config)

showTime 弹幕是否显示时间

  • [弹幕]是否显示时间? true 是 / false 否
    建议为 false
  • 如果设置为 true 那么控制台会显示成这样: 
      [2014-01-01 00:00:00] ...

showWatcherNum 是否显示在线人数

  • [服务器]是否显示在线人数? true 是 / false 否
    心跳包返回数据后显示。心跳包频率为 10s/次。
    建议为 false (除非需要实时观察人数)
  • 如果设置为 true 那么控制台会显示成这样: 
    #在线人数: 999

showUserName 弹幕是否显示发布者

  • [弹幕]是否显示发布者?(仅限live有效) true 是 / false 否
    建议为 true
  • 如果设置为 true 那么控制台会显示成这样: 
     ... ⑨bishi : ...

reconnect 是否断线重连

  • [服务器]是否断线重连? true 是 / false 否
    建议为 true (一般来说用不上,以防万一)

save 是否保存弹幕源数据

  • [弹幕]是否保存弹幕源数据? true 是 / false 否
    建议为 false
  • 弹幕源数据保存至 程序根目录下的 comments 文件夹,命名格式 <cid>_<timestamp>.source
    具体弹幕源数据的格式请看弹幕源数据开发文档。

直播间(exports.liveroom)

概述

  • 本设置为必填项目，用于访问弹幕服务器。有url / roomid两个参数。
  • 只需要选择一个参数设置就可以访问弹幕服务器了。
  • 下面会讲到二者差别。

url 直播室网址

  • 直播室网址
    例如: url:'http://live.bilibili.com/live/1029.html' (切记 一定要用 英文逗号圈住地址)
  • 这里是范本: 
    exports.liveroom={
    url:'http://live.bilibili.com/live/1029.html'
};

roomid 直播室id

  • 直播室id
    例如 roomid:'1001' 或者 roomid:1001
  • 这里是范本: 
    exports.liveroom={
    roomid:1001
};
